Configuring Git Shell

Open Git Shell (command line) and begin with the following commands:

$ git config --global user.name "John Doe"
$ git config --global user.email yourEmail@example.com

mention $ git config --list to verify information

or Use GUI to do configuration - this is in the settings, then options (see original tutorial when explaining and use image from there)

What is a repo (aka repository)

How to create a repo

ease to do so from online version to later clone with images

Command: $ git init initializes a git repository. This creates the initial ‘.git’ directory in a new or in an existing project pending on what directory the user precedes the command with. Example: $ /home/user/my_new_git_folder/git init. In this example, the portion before the command git init designates a local file path.

links for command line setup https://www.kernel.org/pub/software/scm/git/docs/user-manual.html#creating-a-new-repository and https://git-scm.com/book/en/v2/Git-Basics-Getting-a-Git-Repository

Basic Git workflow:

  1. You modify files in your working directory.
  2. You stage the files, adding snapshots of them to your staging area. This is done with the command git add.
  3. You do a commit, which takes the files as they are in the staging area and stores that snapshot permanently to your local Git directory. This is done with the command git commit.
  4. You push local changes to GitHub server, with the command git push.
